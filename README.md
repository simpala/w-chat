# Artifacts Panel API

## Overview

This document describes the Go backend API for the "Artifacts Panel" in the Wails application. The API is designed to manage non-textual content generated by LLMs or tools, such as images, videos, and notifications.

The system is designed to be modular and extensible, allowing new types of artifacts to be added with minimal changes.

**Core Concepts:**
- **Artifact:** A piece of generated content (e.g., an image, a tool notification).
- **ArtifactService:** A Go service that manages the lifecycle of artifacts.
- **Persistence:** Artifacts can be marked as `persistent` or `non-persistent`.
  - **Persistent artifacts** (e.g., images, videos) have their content saved to disk and are reloaded when the application starts. Their metadata, however, is currently only stored in memory.
  - **Non-persistent artifacts** (e.g., tool notifications) exist only in memory and are discarded when the session ends or the application closes.

## Setup and Integration

To use this API in your Wails project, follow these steps:

1.  **Place Files**: Copy the `artifacts` directory into your project's Go source tree.
2.  **Integrate into `main.go`**: You need to initialize and bind the `ArtifactService` in your main application struct and `wails.Run` configuration.

See the `main.go.snippet` file for a detailed example. The key steps are:
-   Import the `artifacts` package.
-   Add `*artifacts.ArtifactService` to your `App` struct.
-   In your `OnStartup` method:
    -   Define a `sessionID`.
    -   Define the `artifactsDir` where content files will be stored. A good location is a subdirectory within `os.UserConfigDir()`.
    -   Create a new instance of the `ArtifactService` using `artifacts.NewArtifactService(...)`.
    -   Assign the new instance to your `App` struct.
    -   Use `defer artifactService.CleanupNonPersistentArtifacts(sessionID)` to ensure temporary artifacts are cleared on exit.
-   In the `wails.Run` options, add your `ArtifactService` instance to the `Bind` slice. This makes its public methods available to the frontend.

```go
// In your wails.Run options:
Bind: []interface{}{
    app,
    app.ArtifactService, // <-- Add this
},
```

## How to Access the API from the Frontend

Once the `ArtifactService` is bound, Wails automatically generates TypeScript bindings. You can call its methods from your frontend JavaScript/TypeScript code.

**Default Location of Bindings:** `frontend/wailsjs/go/artifacts/ArtifactService.js`

### 1. Calling Go Methods

Import the generated module and call the methods as if they were local async functions.

```typescript
// Import the Go service methods
import * as GoArtifacts from '../../wailsjs/go/artifacts/ArtifactService';

// Example: Get a list of all persistent artifacts on load
async function loadArtifacts(sessionID: string) {
    try {
        const artifacts = await GoArtifacts.ListArtifacts(sessionID);
        console.log('Loaded artifacts:', artifacts);
        // ...render the artifacts
    } catch (err) {
        console.error('Failed to load artifacts:', err);
    }
}

// Example: Deleting an artifact via a button click
async function handleDelete(artifactId: string) {
    try {
        await GoArtifacts.DeleteArtifact(artifactId);
        console.log(`Successfully requested deletion for artifact: ${artifactId}`);
        // The frontend will receive an 'artifactDeleted' event to update the UI
    } catch (err) {
        console.error('Failed to delete artifact:', err);
    }
}

// Example: Adding a new artifact (less common from frontend, but possible)
async function createNotification(message: string) {
    const artifactType = "TOOL_NOTIFICATION";
    const content = null; // No file content for notifications
    const metadata = { message: message };
    const isPersistent = false; // Notifications are temporary

    try {
        const newArtifactId = await GoArtifacts.AddArtifact(artifactType, content, metadata, isPersistent);
        console.log(`Created new notification artifact with ID: ${newArtifactId}`);
        // The frontend will receive a 'newArtifactAdded' event
    } catch (err) {
        console.error('Failed to create notification:', err);
    }
}
```

### 2. Listening for Go Events

The `ArtifactService` emits events to the frontend when the state changes. This is the primary way to keep the UI in sync in real-time.

```typescript
// Import the Wails runtime event listener
import * as GoRuntime from '../../wailsjs/runtime/runtime';

// Listen for a new artifact being added
GoRuntime.EventsOn("newArtifactAdded", (artifactID: string) => {
    console.log(`A new artifact was added with ID: ${artifactID}. Fetching details...`);
    // Now, call GetArtifact(artifactID) to get the data and render it
});

// Listen for an artifact being deleted
GoRuntime.EventsOn("artifactDeleted", (artifactID:string) => {
    console.log(`Artifact with ID ${artifactID} was deleted.`);
    // Find and remove the artifact's element from the DOM
    const elem = document.getElementById(`artifact-${artifactID}`);
    if (elem) {
        elem.remove();
    }
});
```

## How to Implement a New Artifact Type

Follow these steps to add support for a new kind of artifact (e.g., `CODE`).

#### Step 1: Define the New ArtifactType

In `artifacts/types.go`, add a new constant for your artifact type.

```go
// artifacts/types.go
const (
    // ... existing types
    TypeCode ArtifactType = "CODE" // <-- Add this
)
```

#### Step 2: Update `AddArtifact` Logic (If Needed)

In `artifacts/service.go`, modify the `AddArtifact` method if your new type has special requirements.

-   **If it needs file storage**: Add your new `TypeCode` to the `switch artifactType` block to handle saving the `content` bytes to a file. Choose an appropriate file extension.
-   **If it's metadata-only**: You might not need any changes. The `metadata` map can hold all the necessary information. For a `CODE` artifact, you might store the language in the metadata: `{"language": "python"}`.

#### Step 3: Update `DeleteArtifact` Logic (If Needed)

The current `DeleteArtifact` logic automatically handles file deletion for any artifact that is `IsPersistent` and has a `ContentPath`. No changes should be needed unless your new type requires special cleanup.

#### Step 4: Implement Frontend Rendering

In your frontend code, you must add logic to render the new artifact type.

In the function that creates HTML elements for artifacts (`createArtifactElement` in the example), add a new `case` to the `switch` statement.

```typescript
// frontend_example.ts

function createArtifactElement(artifact: GoArtifacts.Artifact): HTMLElement | null {
    // ...
    switch (artifact.type) {
        // ... existing cases
        case "CODE":
            contentElement = document.createElement('pre');
            const code = document.createElement('code');
            // Assuming code content is stored in metadata for smaller snippets
            code.textContent = artifact.metadata['code_snippet'] || '...';
            // You might want to use a syntax highlighting library here
            code.className = `language-${artifact.metadata['language']}`;
            contentElement.appendChild(code);
            break;
    // ...
    }
    // ...
}
```

By following these steps, you can cleanly extend the system to support any new type of generated content.
